Welcome to the HDF 2.0 Streaming Analytics Reference Application

The specific components in this architecture are the following:

For the Use Details, check out the HDF Streaming Anlaytics Reference Application

  • Minifi - Collect Events from the Edge. The edge here represents collecting events from the different sensors on the truck
  • Nifi - Performs site to site data movement, intelligent data routing, transformations, enrichment, and delivers the streams to the stream processing engine.
  • Kafka - Scalable Event Broker that acts as the distributed storage Substrate for the Stream Processing engine
  • Storm - Streaming analytics engine used to build a stream application demonstrating joining streams, windoweed aggreation, fast real-tile model scoring, serving requests to Dashboard
  • HBase - Store the Time-Series Trucking Events for Fast Access
